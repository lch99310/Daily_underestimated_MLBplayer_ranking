How It Works

wOBA

Weighted On-Base Average assigns linear weights to all offensive events — walks, singles, doubles, triples, home runs — based on their empirical run value. Unlike batting average, wOBA captures the true offensive contribution of each plate appearance.

wOBA = (0.69*BB + 0.89*1B + 1.27*2B + 1.62*3B + 2.10*HR) / PA

xwOBA

Expected wOBA uses Statcast data — exit velocity and launch angle — to predict what a player's wOBA should be based on quality of contact, removing the influence of luck, defensive alignment, and park effects.

xwOBA = f(exit_velocity, launch_angle) per batted ball event

diff_rolling_OBA

Measures the gap between rolling wOBA and xwOBA over a configurable plate-appearance window (50/100/150 PA). Negative values indicate a player whose results trail their contact quality — the hallmark of an underestimated player whose surface stats don't reflect true ability.

diff_rolling_OBA = rolling_wOBA(N PA) − rolling_xwOBA(N PA)

Data Pipeline

Real pitch-level Statcast data is fetched from Baseball Savant via the pybaseball library. For each batter, every plate appearance's woba_value and estimated_woba_using_speedangle are used to compute true rolling windows with no approximations.

Baseball Savant → pybaseball → Rolling Windows → JSON → Dashboard
